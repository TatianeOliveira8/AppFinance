## Legenda

**Prioridade**

| Símbolo | Nível | Critério |
|---------|-------|----------|
| 🔴 Alta | Must have | Funcionalidade essencial para o MVP ou requisito obrigatório do cliente |
| 🟡 Média | Should have | Agrega valor significativo mas não bloqueia o uso do app |
| 🟢 Baixa | Nice to have | Melhoria de experiência, pode ser adiada sem impacto crítico |

**Estimativa de esforço (Planning Poker — Story Points)**

| Pontos | Esforço | Referência |
|--------|---------|------------|
| 2 pts | Muito baixo | Tela simples, sem lógica de negócio |
| 3 pts | Baixo | Funcionalidade direta com uma tela e endpoints básicos |
| 5 pts | Médio | Envolve múltiplas telas, validações ou integrações simples |
| 8 pts | Alto | Lógica complexa, múltiplas entidades ou integração com sensor/serviço externo |
| 13 pts | Muito alto | Alta complexidade técnica, parsing de arquivos e lógica de conciliação |
